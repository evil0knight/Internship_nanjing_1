## 1.嘉立创导入AD封装

### 方式一：单个原件“散买”法

* **适用场景** ：板子画到一半，发现少了一个座子或电容，临时去补货。
* **详细步骤** ：

1. 在立创商城找原器件，点击数据手册，然后选择打开。
2. 这时候会进入嘉立创的网页版
3. 点击  **文件 -> 导出 -> Altium Designer** ，原理图和PCB都要。
4. 把导出的两个文件放到工程里。
5. PCB就点击菜单  **Design -> Make PCB Library** ，把这一个封装提取出来，原理图直接复制粘贴。
6. 关联一下封装，双击原理图

---

### 方式二：全板原件“批发”法

* **详细步骤** ：

1. **进货（立创 EDA 端）** ：
   * 在立创 EDA 里新建一个空白工程。
   * 在库里搜出所有要用的零件（如 XT30、GH1.25、电阻电容等），通通丢到原理图画布上。
2. **打包（转 PCB）** ：
   * 点击顶部菜单的  **设计 (Design) -> 原理图转 PCB** （或者更新 PCB）。
   * 这时候你会看到所有零件的封装都整齐地排在黑色 PCB 区域。
3. **过海关（导出工程）** ：
   * 在 PCB 界面，点击  **文件 -> 导出 -> Altium Designer** 。
   * 下载后你会得到一个完整的 AD 工程压缩包或文件。
4. **拆解入库（AD13 端）** ：
   * 用 **AD13** 打开这个导出的 PCB 文件。
   * 点击顶部菜单  **Design -> Make PCB Library** 。
   * **奇迹时刻** ：AD 会自动扫描这张图上所有的零件，并把它们**全部**提取出来，生成一个包含所有零件的 `.PcbLib` 库文件。
   * 然后关联一下

### 2. 中文丝印显示乱码或方框

* **现象** ：在 PCB 上写“滑环转接板”，结果显示成 `乱码`。
* **解决** ：将字体从矢量线条改为系统自带字库。
* **详细步骤** ：

1. **双击**那个乱码文字。
2. 在弹出的属性框右侧 **Font** 区域，点击选中 **TrueType** 圆点。
3. 在下方的 **Font Name** 下拉菜单里，手动找到并选择 **“微软雅黑”** 或  **“SimSun”** 。
4. 检查 **Layer** 选项，必须是  **Top Overlay** （黄色）或  **Top Silkscreen** 。
5. 点击  **OK** 。

---

## 3. PCB 上总出现删不掉的红色大方块

* **现象** ：每次更新原理图，板子上都会自动多出一个红色半透明方块，挡住视线。
* **解决** ：在项目设置里关掉这个“自动生成房间”的功能。
* **详细步骤** ：

1. 在顶部菜单点击  **Project -> Project Options** 。
2. 切换到 **Class Generation** 标签页。
3. 找到 **Component Classes** 这一行，取消勾选对应的  **Generate Rooms** （通常是最后一列）。
4. 点击  **OK** 。
5. 回到 PCB，**手动点中**那个红色方块，按 **Delete** 键最后删它一次，它就再也不会回来了。

---

### 4. 底部图层标签太多，找不到黄色层

* **现象** ：屏幕底部有一排五颜六色的标签，把想找的 Top Overlay 挤没了。
* **解决** ：隐藏不用的机械层。
* **详细步骤** ：

1. 按下键盘快捷键  **L** 。
2. 在弹出的 `View Configurations` 窗口右侧，找到  **Mechanical Layers** 。
3. 把除了 `Mechanical 1`（边框层）之外的所有层的 **Enable** 和 **Show** 勾选都点掉。
4. 点击窗口下方的  **Used Layers On** （只显示用到的层）。
5. 点击  **OK** ，底部就会变清爽了。

---

### 5. 焊盘对齐与等间距排列

* **现象** ：几个座子高低不一，左右距离也靠眼睛瞄，很不专业。
* **解决** ：使用对齐快捷键。
* **详细步骤** ：

1. 鼠标左键拉个框，**选中**那几个座子。
2. **下对齐** ：按下键盘  **A** ，再按下  **B(botton)** 。
3. **左右等距** ：按下键盘  **A** ，再按下  **D(distribute horizontally)** 。

---

### 6. 布线满屏绿叉 (DRC 报错)（待定）

* **现象** ：一画线就变绿，满屏都是交叉的小符号。
* **解决** ：放宽系统对“安全距离”的要求。
* **详细步骤** ：

1. 按下键盘快捷键 **D + R** (Design Rules)。
2. 在左侧树状图找到  **Electrical -> Clearance** ，双击它。
3. 把右边的 **Minimum Clearance** 从默认的 **$10\text{mil}$** 改小（比如改成 **$6\text{mil}$** 或  **$0.2\text{mm}$** ）。
4. 点击  **OK** 。
5. **刷新显示** ：按下键盘  **T + M** ，绿叉就会消失。

---

### 7. 导入立创封装并合并库

* **现象** ：从立创导出的封装是一个独立文件，想把它放进自己的主库里。
* **解决** ：在 PCB Library 面板直接复制。
* **详细步骤** ：
* 打开立创导出的那个 `.PcbLib` 文件。
* 点开右下角的 **PCB -> PCB Library** 面板。
* 在列表里**右键**点击那个封装名字，选  **Copy** 。
* 在左侧项目栏切换回你自己的主库（比如 `PCB1.PcbLib`）。
* 在同样的 **PCB Library** 面板空白处，**右键**选  **Paste** 。
* 按 **Ctrl + S** 保存。

### 一、日常推送 / 拉取代码（核心流程）

只要仓库已经关联了 SSH 地址，后续每次修改代码后，执行以下 3 步即可（和之前 HTTPS 方式几乎一样，只是不用输密码）：

bash

运行

```
# 1. 拉取远程最新代码（避免冲突，可选但推荐）
git pull

# 2. 添加所有修改的文件到暂存区
git add .

# 3. 提交变更（备注写清楚修改内容）
git commit -m "修复XX问题 / 新增XX功能"

# 4. 推送到 GitHub（无需输密码，SSH 自动验证）
git push
```

### 二、给已有仓库切换 / 确认 SSH 地址（关键）

如果是之前用 HTTPS 地址的仓库，需要先把远程地址换成 SSH 格式（只需要做一次）：

bash

运行

```
# 1. 查看当前远程仓库地址（确认是否是 SSH 格式）
git remote -v

# 2. 如果显示 https:// 开头，执行下面命令切换为 SSH 地址
git remote set-url origin git@github.com:evil0knight/Internship.git
# 替换规则：GitHub SSH 地址格式是 git@github.com:用户名/仓库名.git

# 3. 再次查看，显示 git@github.com:... 就是切换成功了
git remote -v
```

### 三、新建仓库时直接用 SSH（一劳永逸）

以后在 GitHub 新建仓库后，复制「SSH 地址」而非 HTTPS 地址，本地克隆 / 关联时直接用：

bash

运行

```
# 克隆新建的 GitHub 仓库到本地（SSH 方式）
git clone git@github.com:evil0knight/新仓库名.git

# 进入克隆后的文件夹，直接开发、提交、推送即可，无需额外配置
cd 新仓库名
```

### 四、验证 SSH 连接是否有效（出问题时排查）

如果推送时提示 SSH 验证失败，先执行这个命令测试连接：

bash

运行

```
ssh -T git@github.com
```

* 成功：会提示 `Hi evil0knight! You've successfully authenticated...`
* 失败：根据提示排查（比如密钥没加对、防火墙拦截等）

### 五、常见问题处理

1. **换设备 / 重装系统** ：在新设备上重新生成 SSH 密钥（重复之前的 `ssh-keygen` 步骤），并把新公钥添加到 GitHub 即可；
2. **密钥失效 / 丢失** ：在 GitHub 「SSH and GPG keys」页面删除旧密钥，重新生成 + 添加新密钥；
3. **多账号区分** ：如果有多个 GitHub 账号，可配置 `~/.ssh/config` 文件区分密钥（新手暂时不用管）。
