## 1.嘉立创导入AD封装

### 方式一：单个原件“散买”法

* **适用场景** ：板子画到一半，发现少了一个座子或电容，临时去补货。
* **详细步骤** ：

1. 在立创商城找原器件，点击数据手册，然后选择打开。
2. 这时候会进入嘉立创的网页版
3. 点击  **文件 -> 导出 -> Altium Designer** ，原理图和PCB都要。
4. 把导出的两个文件放到工程里。
5. PCB就点击菜单  **Design -> Make PCB Library** ，把这一个封装提取出来，原理图直接复制粘贴。
6. 关联一下封装，双击原理图

---

### 方式二：全板原件“批发”法

* **详细步骤** ：

1. **进货（立创 EDA 端）** ：
   * 在立创 EDA 里新建一个空白工程。
   * 在库里搜出所有要用的零件（如 XT30、GH1.25、电阻电容等），通通丢到原理图画布上。
2. **打包（转 PCB）** ：
   * 点击顶部菜单的  **设计 (Design) -> 原理图转 PCB** （或者更新 PCB）。
   * 这时候你会看到所有零件的封装都整齐地排在黑色 PCB 区域。
3. **过海关（导出工程）** ：
   * 在 PCB 界面，点击  **文件 -> 导出 -> Altium Designer** 。
   * 下载后你会得到一个完整的 AD 工程压缩包或文件。
4. **拆解入库（AD13 端）** ：
   * 用 **AD13** 打开这个导出的 PCB 文件。
   * 点击顶部菜单  **Design -> Make PCB Library** 。
   * **奇迹时刻** ：AD 会自动扫描这张图上所有的零件，并把它们**全部**提取出来，生成一个包含所有零件的 `.PcbLib` 库文件。
   * 然后关联一下

### 2. 中文丝印显示乱码或方框

* **现象** ：在 PCB 上写“滑环转接板”，结果显示成 `乱码`。
* **解决** ：将字体从矢量线条改为系统自带字库。
* **详细步骤** ：

1. **双击**那个乱码文字。
2. 在弹出的属性框右侧 **Font** 区域，点击选中 **TrueType** 圆点。
3. 在下方的 **Font Name** 下拉菜单里，手动找到并选择 **“微软雅黑”** 或  **“SimSun”** 。
4. 检查 **Layer** 选项，必须是  **Top Overlay** （黄色）或  **Top Silkscreen** 。
5. 点击  **OK** 。

---

## 3. PCB 上总出现删不掉的红色大方块

* **现象** ：每次更新原理图，板子上都会自动多出一个红色半透明方块，挡住视线。
* **解决** ：在项目设置里关掉这个“自动生成房间”的功能。
* **详细步骤** ：

1. 在顶部菜单点击  **Project -> Project Options** 。
2. 切换到 **Class Generation** 标签页。
3. 找到 **Component Classes** 这一行，取消勾选对应的  **Generate Rooms** （通常是最后一列）。
4. 点击  **OK** 。
5. 回到 PCB，**手动点中**那个红色方块，按 **Delete** 键最后删它一次，它就再也不会回来了。

---

### 4. 底部图层标签太多，找不到黄色层

* **现象** ：屏幕底部有一排五颜六色的标签，把想找的 Top Overlay 挤没了。
* **解决** ：隐藏不用的机械层。
* **详细步骤** ：

1. 按下键盘快捷键  **L** 。
2. 在弹出的 `View Configurations` 窗口右侧，找到  **Mechanical Layers** 。
3. 把除了 `Mechanical 1`（边框层）之外的所有层的 **Enable** 和 **Show** 勾选都点掉。
4. 点击窗口下方的  **Used Layers On** （只显示用到的层）。
5. 点击  **OK** ，底部就会变清爽了。

---

### 5. 焊盘对齐与等间距排列

* **现象** ：几个座子高低不一，左右距离也靠眼睛瞄，很不专业。
* **解决** ：使用对齐快捷键。
* **详细步骤** ：

1. 鼠标左键拉个框，**选中**那几个座子。
2. **下对齐** ：按下键盘  **A** ，再按下  **B(botton)** 。
3. **左右等距** ：按下键盘  **A** ，再按下  **D(distribute horizontally)** 。

---

### 6. 布线满屏绿叉 (DRC 报错)（待定）

* **现象** ：一画线就变绿，满屏都是交叉的小符号。
* **解决** ：放宽系统对“安全距离”的要求。
* **详细步骤** ：

1. 按下键盘快捷键 **D + R** (Design Rules)。
2. 在左侧树状图找到  **Electrical -> Clearance** ，双击它。
3. 把右边的 **Minimum Clearance** 从默认的 **$10\text{mil}$** 改小（比如改成 **$6\text{mil}$** 或  **$0.2\text{mm}$** ）。
4. 点击  **OK** 。
5. **刷新显示** ：按下键盘  **T + M** ，绿叉就会消失。(错误我怀疑AD13有问题，准备装17)

---

### 7. 导入立创封装并合并库

* **现象** ：从立创导出的封装是一个独立文件，想把它放进自己的主库里。
* **解决** ：在 PCB Library 面板直接复制。
* **详细步骤** ：
* 打开立创导出的那个 `.PcbLib` 文件。
* 点开右下角的 **PCB -> PCB Library** 面板。
* 在列表里**右键**点击那个封装名字，选  **Copy** 。
* 在左侧项目栏切换回你自己的主库（比如 `PCB1.PcbLib`）。
* 在同样的 **PCB Library** 面板空白处，**右键**选  **Paste** 。
* 按 **Ctrl + S** 保存。

**第一步：建表**

定义状态和事件的枚举，再搞个状态转移表：

```c
// 状态和事件
typedef enum { IDLE, RUNNING, ERROR } State;
typedef enum { START, STOP, ERROR_EVENT } Event;

// 表结构
typedef struct {
    State current;  // 当前状态
    Event event;    // 触发事件
    State next;     // 下一状态
    void (*action)(void); // 执行的操作
} Transition;

// 状态转移表
Transition transitions[] = {
    {IDLE, START, RUNNING, start_running},
    {RUNNING, STOP, IDLE, stop_running},
    {RUNNING, ERROR_EVENT, ERROR, handle_error},
    // 加新状态？在这加一行
};
```

**第二步：查表+执行**

写个函数，根据当前状态和事件查表，更新状态并干活：

```c
void process_event(Event event) {
    int size = sizeof(transitions) / sizeof(Transition);
    for (int i = 0; i < size; i++) {
        if (transitions[i].current == current_state && transitions[i].event == event) {
            current_state = transitions[i].next;
            if (transitions[i].action) transitions[i].action();
            return;
        }
    }
    invalid_transition(); // 没找到就报个错
}
```
